This is for JM0250 Data Visualization. Prioritize learning and course-compliant design.

## Visualization design rules
- Use Munzner's What/Why/How framework.
- Justify visual encodings using expressiveness and effectiveness.
- Discuss marks, channels, interaction, scalability, and task support.
- Prefer linked views and interaction where useful.
- Avoid unjustified 3D, chart junk, and misleading encodings.

## Coding rules

- Do not rewrite the whole project unless explicitly asked.
- After code changes, explain which files changed and why.
- Prefer readable code over clever code.
