## Abstract requirements

The abstract must summarize:

- Domain problem.
- Data source.
- Main design: coordinated views, scatterplots, ternary plot, dotplot, coverage matrix.
- Main interaction: filter/select/connect/details-on-demand.
- Evaluation/use cases.
- Limitation: exploratory association, not causality.

### Tourism indicator justification

Add:

```latex
International tourism receipts as a percentage of total exports was selected as the main tourism-dependence indicator because it normalizes tourism revenue by a country's export structure. This makes the measure more comparable across countries of different absolute economic size than raw tourism receipts or visitor arrivals. Alternative tourism indicators, such as international arrivals or absolute tourism receipts, capture tourism volume but not necessarily dependence. The limitation is that this single indicator does not capture domestic tourism or indirect employment generated by tourism; therefore, the dashboard supports exploratory comparison rather than a complete causal analysis of tourism.
```

### Derived attributes and formulas

Use these formulas if they match the code. If not, update them to match code.

```latex
\textbf{Latest available value.}
For country $c$, year $y$, and attribute $a$, the latest-available value is defined as the most recent non-missing value observed at or before $y$:
\[
v_{latest}(c,y,a)=v(c,t,a), \quad t=\max\{t' \leq y \mid v(c,t',a) \text{ is available}\}.
\]

\textbf{Data coverage.}
Coverage is computed as the share of years with non-missing values for a country and indicator:
\[
coverage(c,a)=\frac{|\{y \mid v(c,y,a) \text{ is available}\}|}{|\{y \mid y \text{ is in the selected time range}\}|}.
\]

\textbf{Structural employment shift.}
For country $c$ and employment sector $s$, structural shift is computed as the latest available employment share minus the earliest available employment share:
\[
shift(c,s)=v(c,y_{last},s)-v(c,y_{first},s).
\]
```

For gender average, verify code first:

```latex
\textbf{Gender-average employment.}
If sector-level total employment is unavailable or if a gender-average mode is selected, the dashboard computes the arithmetic mean of the male and female employment shares for the same country, year, and sector.
```

Only include this if true.

## Design section requirements

For every view, include:

1. What data it shows.
2. Which task(s) it supports.
3. Marks/channels.
4. Why those choices are effective.
5. Alternatives/trade-offs.
6. Limitations.

### Association views

Must say:

- Points = countries.
- X/Y position = quantitative values.
- Hue = region because region is categorical.
- Marker size should be constant unless it has a clear task.
- Trend line is exploratory only.
- Supports T1/T2/T3.
- Overplotting is limited by selected-country filtering; if many countries are shown, transparency/filtering may be needed.
- Small multiples reduce memory burden when comparing multiple variables.

### Ternary trajectory

Must say:

- Shows employment composition, not tourism directly.
- It is linked to tourism through focus-country selection from scatterplots.
- The three values sum to 100%.
- Exact values are available in hover.
- It supports T4.
- It is not ideal for precise lookup or multi-country trajectory comparison.
- Structural-shift dotplot compensates for cross-country change comparison.

### Structural shift dotplot

Must say:

- Shows first-to-latest employment-sector change.
- Shared zero baseline supports signed comparison.
- Better than stacked bars for positive/negative change because stacked bars imply part-to-whole totals and unaligned baselines.
- Supports T5.

### Coverage matrix

Must say:

- Supporting reliability task S1.
- Same selected countries/indicators as other views.
- Helps users interpret whether patterns are trustworthy.
